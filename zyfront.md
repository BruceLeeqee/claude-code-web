# 项目开发规范汇总

## 1. 代码提交规范

- **禁止自动提交代码**：所有提交必须手动确认。
- **提交信息要求**：
  - 使用中文，避免乱码（推荐 CMD + `chcp 65001` 或 PowerShell UTF‑8 文件方式）。
  - 提交类型：`feat`、`fix`、`docs`、`refactor`、`chore`。
  - 后面加(模块名称)：feat(模块名称)：提交功能概述
  - 例子：feat(workbench): 保留非 Splitter 升级并优化标签溢出交互
  - 格式必须包含以下小节，每节至少一条：
    - 【背景】- 为什么要改
    - 【本次改动】- 关键改动点（3‑8 条）
    - 【影响范围】- 模块/页面/任务
    - 【验证】- 编译、测试、截图或回归结果
    - 【风险与回滚】- 潜在风险与回滚方式

- **多仓库提交流程**：
  1. 分别在 `zyback` / `zyfront` / `zytrader` 子仓生成详细提交信息并推送。
  2. 在根仓更新子模块指针并提交。
  3. 返回每个仓库的 commit id 与推送分支。
- **临时文件处理**：提交后删除生成的 `commit_msg.txt` 文件。
- **计划文档存储**：最终构建的计划文件保存到 `./docs` 目录。

## 2. 后端开发规范

### 2.1 API 路径
- 后端接口 **不要** 带 `/api` 前缀。
- 前端请求 **必须** 带 `/api` 前缀。

### 2.2 Controller 层
- 参数使用实体类，命名以 `param` 为后缀。
- 返回类型使用实体类，命名以 `vo` 为后缀。
- **不进行判空**，直接赋值；特殊字段（如金额需除以 100）单独处理。

### 2.3 Service / Mapper / DAO 层
- 所有传参均使用实体类。
- **MyBatis-Plus 改为原生 MyBatis** 用法。

### 2.4 实体类规范
- 所有数据库表对应实体类放在 `vo` 目录下。
- 驼峰命名字段（如 `iCode`、`iName`、`assetCode`、`opType`、`useCash`）**必须**使用 `@JsonProperty` 注解明确 JSON 字段名，防止序列化后变成小写。

  ```java
  @JsonProperty("iCode")
  private String iCode;
