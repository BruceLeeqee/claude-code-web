# Git 提交并推送到远程

## 触发场景
用户说「提交到远程」「推送到远程」「git push」时触发。

## 描述
将当前分支的更改提交并推送到远程仓库。

## 执行步骤

### 1. 检查 Git 状态
```bash
git status --short
```
若有未提交更改，继续下一步。

### 2. 查看更改内容（可选）
```bash
git diff --stat
```

### 3. 提交更改
```bash
git add -A
git commit -m "<提交信息>"
```

### 4. 推送到远程
```bash
git push origin <当前分支名>
```
或简写：
```bash
git push
```

## 注意事项
- 若远程分支领先，提示用户先拉取合并
- 提交信息应简洁说明「为什么要改」
- 推送前确认分支名正确
