## 1. 系统简介

### 1.1 什么是多智能体系统？

多智能体系统是一个智能任务执行框架，能够根据任务复杂度自动选择最优执行方式：

- **简单任务**：由单个Agent快速完成
- **复杂任务**：自动拆分并分配给多个专业Agent协作完成

### 1.2 核心优势

| 特性 | 说明 |
|------|------|
| 🚀 智能决策 | 自动判断任务复杂度，选择最优执行模式 |
| 🔄 灵活切换 | 支持手动切换单Agent/多智能体模式 |
| 👥 角色分工 | 自动分配研究员、规划者、执行者等专业角色 |
| 📊 可视化 | 实时展示任务拆分和智能体状态 |
| 🔒 安全可控 | 敏感操作需确认，完整审计日志 |

## 2. 快速入门

### 2.1 首次使用

1. 打开应用程序，进入**终端页面**
2. 在右侧边栏可以看到**执行模式指示器**
3. 默认显示"单Agent"模式，表示系统处于自动决策状态

### 2.2 执行第一个任务

**简单任务示例**：

```
输入: 读取README.md文件内容
```

系统会：
1. 分析任务复杂度 → 判定为"简单"
2. 选择单Agent模式
3. 快速执行并返回结果

**复杂任务示例**：

```
输入: 实现一个用户登录系统，包括前端表单、后端API和数据库设计
```

系统会：
1. 分析任务复杂度 → 判定为"复杂"
2. 切换到多智能体模式
3. 自动拆分任务并分配角色
4. 协调多个Agent并行执行
5. 汇总结果返回

### 2.3 查看执行状态

在右侧边栏可以查看：

- **任务列表**：当前任务的拆分情况
- **智能体列表**：活跃的智能体及其状态
- **事件日志**：系统事件和执行记录

## 4. 执行模式

### 4.1 单Agent模式

**适用场景**：
- ✅ 简单查询（如"读取文件内容"）
- ✅ 单文件修改
- ✅ 简单问答
- ✅ 快速测试

**特点**：
- 响应速度快（通常 < 500ms）
- 不拆分任务
- 直接执行返回结果
- 资源消耗低

### 4.2 多智能体模式

**适用场景**：
- ✅ 多模块开发
- ✅ 跨领域任务（前端+后端+测试）
- ✅ 并行任务
- ✅ 复杂系统架构

**特点**：
- 自动任务拆分
- 角色分工协作
- 并行/串行智能调度
- 结果汇总整合

### 4.3 自动决策规则

系统根据以下因素自动选择模式：

| 因素 | 权重 | 说明 |
|------|------|------|
| 任务长度 | 低 | 请求文本长度 |
| 关键词 | 中 | 包含"实现"、"设计"等 |
| 跨领域 | 高 | 涉及多个技术领域 |
| 并行性 | 高 | 可并行执行的任务 |
| 依赖关系 | 中 | 任务间的依赖程度 |
| 技术复杂度 | 高 | 技术难度评估 |
| 业务复杂度 | 中 | 业务逻辑复杂度 |

**决策阈值**：

```
复杂度评分 ≤ 2  → 单Agent模式
复杂度评分 3-5  → 多智能体模式（中等）
复杂度评分 > 5  → 多智能体模式（复杂）
```

### 4.4 强制模式

用户可以手动强制指定执行模式：

1. 点击**切换按钮**强制切换
2. 强制模式会显示特殊标记
3. 点击**重置按钮**恢复自动决策

## 5. 配置设置

### 5.1 打开配置面板

通过配置面板可以调整系统参数。

### 5.2 配置项说明

| 配置项 | 默认值 | 范围 | 说明 |
|--------|--------|------|------|
| 启用多智能体 | 开启 | 开/关 | 控制是否启用多智能体模式 |
| 最大智能体数量 | 5 | 1-10 | 限制同时运行的智能体数量 |
| 执行模式 | 自动 | 自动/单Agent/多智能体 | 默认执行模式 |
| 执行后端 | 进程内 | 进程内/Tmux/iTerm2 | 任务执行环境 |

### 5.3 配置建议

**开发环境**：
- 启用多智能体：开启
- 最大智能体数量：5
- 执行模式：自动

**生产环境**：
- 启用多智能体：开启
- 最大智能体数量：10
- 执行模式：自动

**测试环境**：
- 启用多智能体：开启
- 最大智能体数量：3
- 执行模式：单Agent（便于调试）

### 5.4 恢复默认配置

点击配置面板底部的"恢复默认"按钮即可。

## 6. 使用场景

### 6.1 场景一：代码查询

```
用户输入: 查看src/app目录结构
```

**系统响应**：
- 模式：单Agent
- 原因：简单查询任务
- 执行：直接返回目录结构

### 6.2 场景二：功能开发

```
用户输入: 实现用户注册功能，包括表单验证和API接口
```

**系统响应**：
- 模式：多智能体
- 原因：涉及前端和后端开发
- 任务拆分：
  1. 设计注册表单（前端Agent）
  2. 实现验证逻辑（前端Agent）
  3. 创建API接口（后端Agent）
  4. 编写测试用例（测试Agent）

### 6.3 场景三：系统重构

```
用户输入: 重构用户模块，优化性能并添加单元测试
```

**系统响应**：
- 模式：多智能体
- 原因：复杂重构任务
- 任务拆分：
  1. 分析现有代码（研究员Agent）
  2. 设计重构方案（规划者Agent）
  3. 执行重构（执行者Agent）
  4. 编写测试（验证者Agent）
  5. 代码评审（评审者Agent）

### 6.4 场景四：文档生成

```
用户输入: 为API模块生成文档
```

**系统响应**：
- 模式：单Agent
- 原因：文档生成任务
- 执行：分析代码并生成文档

### 6.5 场景五：全栈开发

```
用户输入: 开发一个完整的博客系统，包括前端、后端和数据库
```

**系统响应**：
- 模式：多智能体
- 原因：跨领域复杂任务
- 角色分配：
  - 研究员：调研技术方案
  - 规划者：设计系统架构
  - 执行者（前端）：开发前端界面
  - 执行者（后端）：开发后端API
  - 执行者（数据库）：设计数据库
  - 验证者：编写测试用例

## 7. 常见问题

### Q1: 为什么简单任务也使用了多智能体模式？

**可能原因**：
1. 任务描述包含复杂关键词
2. 系统配置强制多智能体模式

**解决方案**：
- 点击重置按钮恢复自动决策
- 简化任务描述

### Q2: 多智能体模式响应很慢怎么办？

**可能原因**：
1. 智能体数量过多
2. 任务依赖关系复杂

**解决方案**：
- 减少最大智能体数量配置
- 简化任务描述，明确优先级

### Q3: 如何查看智能体的执行日志？

**方法**：
1. 在右侧边栏切换到"事件"标签
2. 查看事件日志列表
3. 点击具体事件查看详情

### Q4: 敏感操作被阻止怎么办？

**说明**：
系统会自动检测敏感操作（如删除、格式化等），需要用户确认。

**操作**：
1. 查看确认提示
2. 选择"确认执行"或"取消"

### Q5: 如何强制使用单Agent执行复杂任务？

**方法**：
1. 点击模式切换按钮
2. 切换到"单Agent"模式
3. 执行任务

**注意**：复杂任务使用单Agent可能需要更长时间。

### Q6: 智能体状态显示"失败"怎么处理？

**可能原因**：
1. 执行过程中遇到错误
2. 资源不足
3. 权限问题

**解决方案**：
1. 查看错误日志
2. 检查系统资源
3. 确认操作权限

### Q7: 如何重置系统配置？

**方法**：
1. 打开配置面板
2. 点击"恢复默认"按钮
3. 确认重置
