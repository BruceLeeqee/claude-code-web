# Agent Battle Arena UI 优化文档

**文档版本**：1.0  
**目标页面**：`zyfront-desktop/src/app/features/prototype/collaboration/` 多智能体协作页  
**文档目的**：对比原型页与当前实现，梳理差异、定位 UI 问题，并给出按优先级、按里程碑推进的视觉优化方案。

---

## 1. 文档目标

本文档用于回答三个问题：

1. **原型长什么样**：原型页的结构、密度、视觉层级、交互语义是什么。
2. **当前实现差在哪**：与原型相比，哪些地方已经对齐，哪些地方仍有偏差。
3. **下一步怎么优化**：给出明确的 UI 优化方向、优先级、里程碑与验收标准。

本文件偏向**设计评审 + 交付执行**，不是代码说明文档。

---

## 2. 对比范围说明

### 2.1 原型页范围
原型页包含以下视图：

- 竞技舞台
- 协作网络
- 认知实验室
- 系统监控

以及以下通用结构：

- 顶部标题栏
- 左侧工作流时间轴 / 共享看板
- 中央主舞台
- 右侧模式切换 / Agent 状态 / 系统性能
- 底部街机控制条
- 新建 Agent 弹窗
- 创建自定义技能弹窗

### 2.2 当前实现范围
当前实现已经具备上述四个视图和弹窗结构，但在以下方面与原型仍有差距：

- 视觉密度
- 字体风格
- 栅格与边框统一性
- 中央主舞台的留白与节点分布
- 左右面板的信息压缩程度
- 模式页与子模块的图表感
- 弹窗的像素化完成度
- 动效与微反馈的一致性

---

## 3. 原型与当前实现总览对比

### 3.1 已对齐部分
当前实现已经基本对齐原型的核心结构：

- 4 视图切换已完整实现
- 顶部导航与标题区已实现
- 左侧时间轴 / 看板已实现
- 中央战场已实现
- 右侧 Agent 状态与模式选择已实现
- 底部控制条已实现
- 新建 Agent 与自定义技能弹窗已实现

### 3.2 仍存在差异的部分

#### 3.2.1 原型的视觉压缩度更高
原型更像一个“紧凑型像素仪表板”，当前实现更接近“完成度较高的业务型原型页”。

表现为：
- 当前实现间距略大
- 组件边框更规整，但原型更硬、更窄、更像手工拼贴
- 当前实现信息块更现代，原型更复古

#### 3.2.2 原型字体更具像素感
原型页大量使用：
- 低分辨率 HUD 字体感
- 更强的字距压缩
- 更明显的“设备输出感”

当前实现虽然已使用像素风字体思路，但整体仍偏“现代 UI + 像素装饰”。

#### 3.2.3 原型的主舞台更空、更黑、更聚焦
原型的中央舞台：
- 留白更大
- 视觉焦点更集中在少量节点
- 背景几乎完全服从于节点和轨迹

当前实现的舞台内容更完整，但因此少了原型那种“空旷战场”的压迫感。

#### 3.2.4 原型右侧面板更像仪表盘
原型中的右栏更像：
- 运行状态仪表盘
- 资源侧边栏
- 轻量信息卡叠层

当前实现右侧模块虽然结构完整，但块面更均匀，视觉戏剧性稍弱。

## 4. 分区域对比分析

## 4.1 顶部标题栏

### 原型特征
- 标题更小、更紧、更硬
- 右侧视图按钮更像街机标签
- 整体高度较低，像嵌在面板顶部
- 强调“系统控制台”而非导航

### 当前实现
- 标题层级清晰
- 视图切换完整
- 视觉更稳定
- 但按钮和标题的“设备感”不够强

### 优化建议
- 缩小标题副标题字号 10%~15%
- 右侧按钮边框更硬、更窄、更扁
- 增加顶部横向扫描线和噪点纹理
- 压缩标题栏高度 4~8px

---

## 4.2 左侧工作流时间轴 / 共享看板

### 原型特征
- 时间轴更像“事件日志条”
- 每条记录的颜色更纯
- 文字更短，节奏更像战报
- 下方看板是 2×2 小格，像系统面板

### 当前实现
- 时间轴信息完整
- 左栏排版较清晰
- 但事件条的节奏感还可更强

### 优化建议
- 将左栏进一步压缩为固定事件条高度
- 增加更明显的左侧彩色竖线
- 时间戳字号再小一档
- 看板区的卡片间距再收紧
- 让事件语句更短、更像战斗日志

---

## 4.3 中央主舞台

### 原型特征
- 近乎全黑
- 节点少、焦点明确
- 留白极大
- 轨迹与弧线更具装饰性
- 节点不抢主体，像“战场中的少量目标”

### 当前实现
- 主舞台完整度较高
- 节点、弧线、背景层次丰富
- 但视觉信息略多

### 优化建议
- 节点数量控制在 4~5 个核心点
- 将非核心节点默认降对比度
- 中央区域维持更大黑场
- 强化两个弧线与中心点的关系
- 将状态卡固定在左下角，减少舞台内部漂浮物

---

## 4.4 右侧模式选择

### 原型特征
- 模式按钮更像系统模式开关
- 按钮之间留白较大
- 当前态高亮很强
- 图标简单，强调可读性

### 当前实现
- 模式结构较完整
- 状态明确
- 但按钮视觉还略偏现代面板

### 优化建议
- 降低按钮高度
- 提高按钮之间的纵向间距
- 当前激活态提高边框亮度
- 统一 icon 视觉厚度
- 按钮文本更贴近原型大小写风格

---

## 4.5 右侧 Agent 状态

### 原型特征
- 像“成员负载条”而不是表格
- 头像块、姓名、数值、状态一体化
- 颜色和状态映射明确
- 结构非常像系统监控条

### 当前实现
- 已较接近原型
- 但卡片层次仍稍厚

### 优化建议
- 继续减少内部 padding
- 让头像块更方正
- 条形长度与数字更统一
- 状态文字字号更小
- 增强每条的“发光感”而减少“卡片感”

---

## 4.6 系统性能面板

### 原型特征
- 面板较小
- 信息高度压缩
- 更像设备读数
- 不强调图形装饰

### 当前实现
- 指标展示清晰
- 但略偏“概览卡片”

### 优化建议
- 将数值字号再缩小
- 缩减卡片内边距
- 取消过多背景分区
- 让数值呈现更像终端读数

---

## 4.7 协作网络视图

### 原型特征
- 左侧是力导向图
- 右侧是 Sankey
- 下方是工作流时间轴
- 视觉上更“分栏式”，中间留有明显空白

### 当前实现
- 已具备网络图 / Sankey / 时间轴
- 但页面结构还略偏“完整功能页”

### 优化建议
- 网络图与 Sankey 的边框厚度统一
- 网络节点颜色更饱和
- 保留更大的中部黑区
- 让时间轴变得更窄、更像旁路信息

---

## 4.8 认知实验室视图

### 原型特征
- 中央是思维核心图
- 周围有环形虚线轨道
- 右侧是认知指标
- 左下角是技能列表

### 当前实现
- 结构已对齐
- 但环形与虚线轨迹还不够“像仪器”

### 优化建议
- 提高环形虚线精度
- 中心方块缩小 10%~15%
- 认知指标做成更窄的条状模块
- 技能列表边框更强，色块更像插件标签

---

## 4.9 系统监控视图

### 原型特征
- 左侧资源表占大头
- 中部/下部有图表占位
- 右侧是控制台和新建 Agent
- 信息极度模块化

### 当前实现
- 功能上接近
- 但图表占位更完整、偏现代

### 优化建议
- 资源表行高压缩
- 图表占位减少渐变面积
- 右侧控制台做得更窄
- 新建 Agent 按钮更像“插槽”而不是操作按钮

---

## 5. 视觉差异总结

### 5.1 原型更像什么
原型更像：
- 复古街机机台
- 黑底电子战场
- 像素化系统控制面板
- 信息硬编码到屏幕上的设备输出

### 5.2 当前实现更像什么
当前实现更像：
- 完成度更高的产品化原型
- 带有像素风元素的协作控制台
- 结构完整、可用性更强的战场 UI

### 5.3 结论
如果目标是“**产品级可用**”，当前实现已经达到较高水平；
如果目标是“**原型截图级复刻**”，还需要继续压缩、收紧、削减现代感，强化像素设备感。

---

## 6. UI 优化方向

## 6.1 第一优先级：视觉压缩
目标是把页面从“完整面板”往“像素机台”方向收敛。

### 需要做的事
- 缩小外边距与卡片内边距
- 压缩标题栏高度
- 压缩列表项高度
- 减少大块圆角与柔和阴影
- 强化硬边框

### 结果
页面会更像原型，而不是传统后台。

---

## 6.2 第二优先级：字体与字号统一
目标是提升 HUD 感和设备输出感。

### 需要做的事
- 全局采用更统一的像素字体体系
- 强制压缩大标题与副标题的字号差
- 日志、指标、标签统一到更细的字号梯度

### 结果
文字看起来更“像印在机器上的”。

---

## 6.3 第三优先级：主舞台留白重构
目标是增强原型中的“黑场压迫感”。

### 需要做的事
- 减少舞台内部次要信息
- 节点更少、更集中
- 中心战区更空
- 轨迹更轻、更细

### 结果
主舞台会更像原型截图的视觉中心。

---

## 6.4 第四优先级：右栏信息仪表化
目标是让右栏从“卡片集合”转为“仪表盘”。

### 需要做的事
- 模式按钮变窄
- Agent 状态条更硬
- 性能面板更压缩
- 统一条形、数值、状态的排版

### 结果
右侧会更像原型里的运算面板。

---

## 6.5 第五优先级：微动效统一
目标是让页面“活着”，但不打扰阅读。

### 需要做的事
- 节点轻微漂浮
- 状态条脉冲闪烁
- 选中态更亮
- 激活按钮轻微发光
- 在线状态微闪

### 结果
整个页面会更接近原型的电子屏感。

---

## 7. 分阶段优化里程碑

### Milestone M1：像素密度统一
**目标**：把页面从现代 UI 压向像素仪表盘。

#### 任务
- 压缩间距
- 收紧标题栏
- 统一字体字号
- 强化边框

#### 验收
- 页面第一眼更像原型，而不是普通后台。

---

### Milestone M2：舞台焦点重构
**目标**：让中间主舞台更接近截图中的黑场竞技台。

#### 任务
- 减少节点
- 收敛弧线
- 调整节点坐标
- 降低辅助元素存在感

#### 验收
- 中央舞台更空、更聚焦。

---

### Milestone M3：左右栏信息重排
**目标**：把左右栏改成更像原型的系统读数带。

#### 任务
- 左侧日志进一步压缩
- 右侧状态条仪表化
- 看板和性能面板更紧凑

#### 验收
- 左右栏不再像普通卡片区。

---

### Milestone M4：四视图风格统一
**目标**：让四个视图共享同一套像素 UI 语言。

#### 任务
- 统一图表边框
- 统一标题区样式
- 统一按钮、标签、数值规则

#### 验收
- 切换视图时仍像同一台设备。

---

### Milestone M5：动效与交互收尾
**目标**：补齐最后的机械感与反馈感。

#### 任务
- 轻量动画
- 选中态闪烁
- 控制条反馈
- 弹窗开合节奏统一

#### 验收
- 页面具备“原型级完成度”。

---

## 8. 具体改造建议清单

### 8.1 样式类建议
- 减少 `padding`
- 减少 `gap`
- 减少面板圆角或直接取消圆角
- 提高边框对比
- 降低背景装饰层透明度

### 8.2 文字建议
- 标题使用更小字号
- 标签使用更紧的字距
- 数值使用更硬的等宽感
- 状态文本降低视觉权重

### 8.3 布局建议
- 左右栏宽度固定得更严格
- 中央舞台保留更大面积黑场
- 底部控制条压缩高度
- 弹窗增加“像机台界面”的厚重感

---

## 9. 建议的最终视觉目标

最终页面应同时满足三点：

1. **像原型**：结构、配色、密度、气质接近截图。
2. **可用**：交互逻辑和页面组织清晰，能够支持真实业务。
3. **统一**：四个视图、两个弹窗、底部控制条都属于同一套像素系统。

---

## 10. 结论

当前实现已经完成了“从业务页到像素战场原型”的关键跃迁，结构层面基本对齐原型，但在**压缩度、字体硬度、舞台留白、仪表盘感、微动效**方面仍有优化空间。

后续优化应优先围绕以下顺序展开：

1. 压缩视觉密度
2. 收紧字体与间距
3. 重构中央舞台留白
4. 仪表化右侧信息栏
5. 统一四视图动效与反馈

这样可以在保留可用性的同时，将页面继续拉近到原型截图所呈现的“街机像素战场”气质。
