# zyfront-desktop 多智能体底层架构重构修复路线图（重梳版）

> 版本：v1.1  
> 日期：2026-04-26  
> 目标：基于当前已实现的多智能体底座，重新梳理仍需补齐的能力，按优先级形成可执行的修复 TODO 列表。

---

## 1. 当前实现概览

当前项目已经完成了多智能体底座中的大部分“骨架能力”，包括：

- 团队命令入口已接入：`/team-role`、`/team-struct`、`/team-subagent`、`/team-agent`、`/team run struct`
- 角色、结构、运行态、任务、邮箱、日志、恢复、阶段机等核心服务已存在
- workbench 已能转发团队相关指令
- 事件总线与团队事件类型已经比较完整
- 命令层、注册表层、运行时层已形成初步闭环

但从“文档验收”角度看，仍有几类关键问题需要继续推进：

1. **文件化持久化闭环仍需完全稳定**：角色 / 结构 / 团队运行态需要成为真正可恢复、可扫描、可打开的文件资产。
2. **团队隔离仍需收口**：任务板、邮箱、运行态、日志必须按 teamId 彻底隔离，避免全局污染。
3. **hybrid / struct 驱动编排仍需完善**：阶段执行、阶段产物汇总、失败升级、回归验证需要更完整。
4. **UI 联动与可观察性仍需增强**：sidebar / workbench / 状态面板需要更直观地展示 team、struct、stage、任务、消息和日志。
5. **测试与边界场景仍需补齐**：尤其是恢复、清理、并发、多输入解析、异常退出等场景。

## 3. 仍需修复的关键缺口

### 3.1 文件化资产闭环不够完整

虽然已经有文件路径、模板和注册表，但仍要确保：

- 角色 / 结构 / team runtime 都能真正落盘
- 真实文件是唯一可信数据源
- frontmatter 解析与反解析稳定
- 创建、更新、列表、打开、恢复形成闭环
- 资产文件被工作区修改后能同步刷新

### 3.2 团队级隔离需要彻底收口

需要确保：

- `TeamTaskBoardService` 按 `teamId` 分组存储
- `TeamMailboxService` 按 `teamId` 分组存储
- `TeamRuntimeService` 关闭 / 恢复不会影响其他 team
- 所有 team 相关事件都携带完整 team 上下文

### 3.3 编排执行还需更完整

需要进一步补齐：

- struct stage 的真正执行器
- subagent 阶段并行执行
- agent-team 阶段协作推进
- 阶段完成条件判断
- 失败自动升级 / 回退策略
- 回归验证阶段自动触发

### 3.4 UI 联动与观测能力还需增强

需要让 UI 明确展示：

- 当前 team / struct / stage
- 成员状态
- 未读消息
- 任务状态
- 阻塞项与失败原因
- 日志摘要与阶段变更

### 3.5 测试与异常场景还不够

需要补充：

- 命令解析单测
- registry / runtime / mailbox / task board 单测
- 恢复和清理测试
- 并发 team 测试
- 事件 payload 测试

## 5. 修复 TODO 列表

#### TODO-1 文件化资产闭环收口
**目标**：角色、结构、运行态都必须真正以文件形式存在，并可恢复、可编辑、可扫描。

**任务**：

#### TODO-2 团队级隔离收口
**目标**：每个 team 的任务、消息、运行态互不污染。

- `TeamMailboxService` 改为按 `teamId` 分组

- team 相关事件补齐 teamId / runtimeId 上下文

**完成标准**：
- 多 team 并存互不影响
- 关闭某个 team 不影响其他 team

#### TODO-3 清理 / 恢复 / 关闭流程修正

## 7. 关键风险提示

1. **不要继续扩展新命令而不统一解析器**，否则后续维护成本会快速上升。
2. **不要把 team runtime 继续做成全局单例**，多实例和恢复会出问题。
3. **不要让任务 / 消息继续共享全局容器**，团队隔离会失效。
4. **不要先堆 UI 再补文件持久化**，否则前端会依赖不稳定数据源。

---

## 8. 当前结论

当前项目已经具备较完整的多智能体底座骨架，接下来最重要的不是“再开一个新方向”，而是把已有能力收口成稳定闭环：

- 文件化持久化
- 团队隔离
- 命令解析统一
- 清理恢复正确性
- 事件语义一致性

完成这些后，再继续推进 hybrid 编排、文件扫描同步和 UI 联动，整体架构会更稳、更接近最终验收目标。
