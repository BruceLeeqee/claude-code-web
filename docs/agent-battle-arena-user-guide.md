# Agent Battle Arena 用户手册

## 1. 系统概述

Agent Battle Arena 是一个基于 Angular 17+ 的多智能体协作可视化系统，旨在展示智能体之间的协作、竞争和交互过程。系统采用像素风格的 UI 设计，提供了直观的可视化界面，帮助用户理解智能体的行为和决策过程。

### 1.1 系统特点

- **多智能体协作**：支持 6 种不同角色的智能体（架构师、分析师、开发者、测试员、运维、产品）
- **实时可视化**：通过网络图、时间轴、Sankey 图等多种可视化方式展示智能体之间的关系和信息传递
- **多种协作模式**：支持对抗、协作、流水线、脑暴、竞赛 5 种不同的协作模式
- **实时数据流**：通过 WebSocket 实现智能体之间的实时通信和状态更新
- **性能优化**：采用对象池、渲染优化等技术确保系统在高负载下仍能保持流畅

## 2. 功能介绍

### 2.1 主要视图

系统提供了 4 个主要视图，用户可以通过顶部导航栏或按 Tab 键切换：

1. **竞技舞台**：展示智能体之间的对抗和竞争过程，包括实时状态更新和得分变化
2. **协作网络**：通过网络图和 Sankey 图展示智能体之间的关系和信息传递
3. **认知实验室**：展示智能体的认知过程和共享工作区
4. **系统监控**：展示系统性能和资源使用情况，包括 CPU、内存、网络等指标

### 2.2 协作模式

系统支持 5 种不同的协作模式，用户可以通过右侧面板中的模式选择器切换：

1. **对抗模式**：智能体之间进行竞争和辩论
2. **协作模式**：智能体共同解决问题
3. **流水线模式**：智能体按顺序处理任务
4. **脑暴模式**：智能体自由提出想法
5. **竞赛模式**：智能体竞争完成任务

### 2.3 智能体角色

系统包含 6 种不同角色的智能体，每种角色都有其独特的职责和技能：

1. **架构师**：负责系统架构设计和技术指导
2. **分析师**：负责数据分析和市场研究
3. **开发者**：负责代码实现和技术开发
4. **测试员**：负责测试和质量保证
5. **运维**：负责系统部署和监控
6. **产品**：负责产品管理和用户体验

## 3. 使用指南

### 3.1 基本操作

1. **启动系统**：运行 `npm run start` 命令启动开发服务器，然后在浏览器中访问 `http://localhost:4200`
2. **切换视图**：点击顶部导航栏中的按钮或按 Tab 键切换不同视图
3. **切换协作模式**：在右侧面板中点击对应的模式按钮
4. **控制游戏**：
   - 按 P 键：暂停/继续游戏
   - 按 R 键：重置游戏
   - 按 M 键：切换模式
   - 按 Tab 键：循环切换视图
5. **调整速度**：在底部控制面板中选择不同的速度倍率

### 3.2 竞技舞台操作

1. **查看智能体状态**：在右侧面板中查看每个智能体的实时负载和状态
2. **观察得分变化**：在舞台中央查看两个团队的得分变化
3. **查看辩论过程**：在右侧面板中查看智能体之间的辩论内容

### 3.3 协作网络操作

1. **查看网络图**：在左侧面板中查看智能体之间的关系网络
2. **调整网络视图**：使用鼠标拖拽、缩放等操作调整网络视图
3. **查看数据流**：在右侧面板中查看智能体之间的信息传递
4. **生成随机网络**：点击「随机生成」按钮生成随机网络拓扑

### 3.4 认知实验室操作

1. **查看认知视图**：在中央面板中查看智能体的认知过程可视化
2. **查看共享工作区**：在右侧面板中查看智能体的共享工作区和任务
3. **添加项目**：点击「添加项目」按钮添加新的工作项
4. **清除已完成**：点击「清除已完成」按钮清除已完成的工作项

### 3.5 系统监控操作

1. **查看系统负载**：在顶部面板中查看 CPU、内存、网络等系统指标
2. **查看智能体资源使用**：在表格中查看每个智能体的资源使用情况
3. **查看 Token 消耗**：在底部面板中查看 Token 消耗趋势
4. **调整配置**：在配置面板中调整系统配置

## 4. 常见问题

### 4.1 系统启动问题

**问题**：系统启动后页面不显示
**解决方案**：
- 检查开发服务器是否成功启动
- 检查浏览器控制台是否有错误信息
- 确保所有依赖都已正确安装
- 尝试刷新浏览器页面

### 4.2 性能问题

**问题**：系统运行卡顿
**解决方案**：
- 减少同时运行的智能体数量
- 禁用物理引擎（在网络图中）
- 减少粒子效果的数量
- 关闭不必要的浏览器标签页

### 4.3 WebSocket 连接问题

**问题**：WebSocket 连接失败
**解决方案**：
- 确保后端 WebSocket 服务正在运行
- 检查网络连接是否正常
- 检查 WebSocket 服务的 URL 是否正确

### 4.4 功能使用问题

**问题**：某些功能无法正常使用
**解决方案**：
- 检查是否有错误信息显示在浏览器控制台
- 确保所有依赖都已正确安装
- 尝试刷新页面或重新启动系统
