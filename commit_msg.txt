feat(workbench): 新增 DeepSeek 支持、Loop 多任务管理与团队编排

【背景】
- 需要支持 DeepSeek 作为 LLM 提供方，其工具名正则限制要求对 API 层做兼容适配
- Loop 系统需要支持多任务并发管理、团队编排、自动验证矩阵等能力
- Models 页面需要重构以支持新模型配置结构

【本次改动】
- 新增 DeepSeek 工具名清理/还原逻辑（sanitize → restore），兼容 ^[a-zA-Z0-9_-]+$ 限制
- Loop 执行器新增多任务管理与团队编排（teamManager.createTeam/assignSteps）
- Loop 非开发任务（analysis/docs/general）自动标记验证矩阵通过
- Workbench 页面新增 Loop 多视图模式（cards/terminal）、多任务切换与日志缓冲
- Models 页面重构：新增 590 行 UI 逻辑与样式更新，支持新模型配置展示
- 桌面端 main.js 与 preload.js 更新 IPC 桥接能力
- 桌面端模型目录重构（model-catalog.ts），运行时设置同步优化
- App settings 服务跨桌面端/Web 端统一调整

【影响范围】
- zyfront-core：API 层（anthropic-messages、anthropic-stream、api/index、types）
- zyfront-desktop：workbench 模块、models 页面、main/preload 进程、settings 服务
- zyfront-web：settings 页面

【验证】
- 本地编译通过（32 文件变更，2637 行新增，480 行删除）
